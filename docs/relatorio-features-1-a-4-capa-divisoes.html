Biblioteca Universitária API

Relatório Consolidado das Features 1, 2, 3 e 4

Aluno: Leonardo da Silva Sousa

Repositório GitHub: https://github.com/LeonardoSSm/bibliotecaapi

Data: 11/03/2026

Objetivo: consolidar as evidências de atendimento dos requisitos dos PDFs das Features 1 a 4.

Sumário

  • Feature 1 - TDD: Módulo de Reserva de Livros
  • Feature 2 - Integração e Orquestração com APIs externas
  • Feature 3 - Modularização e Design de APIs com DTOs
  • Feature 4 - Autenticação e Autorização com Spring Security

Feature 1 - TDD

Implementação do módulo de reserva com ciclo Red-Green-Refactor, regras de negócio de disponibilidade, duplicidade e cancelamento, com testes unitários em GREEN.

Feature 2 - Integração Externa

Consumo de Google Books e Open Library via OpenFeign, com orquestração de dados internos e externos e cálculo de score de recomendação.

Feature 3 - Multi-módulos + DTOs

Refatoração para arquitetura multi-módulos (common-domain, external-api-client, main-application) e aplicação de DTOs de entrada e saída.

Feature 4 - Segurança

Autenticação HTTP Basic e autorização por roles (ADMIN/USER), com regras por URL/HttpMethod e validação de cenários 401, 403 e acessos permitidos.
